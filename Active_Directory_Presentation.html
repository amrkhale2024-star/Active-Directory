01

Domain و Domain Controller

Domain

كيان منطقي (Logical Entity) يجمع المستخدمين والأجهزة والموارد ضمن شبكة واحدة، وتتم إدارته مركزياً عبر Active Directory.

Domain Controller

جهاز فعلي (Physical Entity) — Windows Server مُثبت عليه دور Active Directory Domain Services (AD DS) ومُرقّى ليصبح Domain Controller.

مثال على اسم Domain

test.local هو اسم Domain يُستخدم عادةً في بيئات الاختبار والتطوير.

Active Directory Domain Services (AD DS) ↓ Promote this server to a Domain Controller ↓ Domain Name: test.local
02

Forest و Forest Root Domain

ما هو Forest؟

الـ Forest هو أعلى مستوى في Active Directory، ويحتوي على Domain واحد أو أكثر. أول Domain يتم إنشاؤه داخل Forest يسمى Forest Root Domain.

┌─────────────────────────────────────────┐ │ Forest │ │ │ │ test.local │ │ (Forest Root Domain) │ │ │ │ ┌──────────┼──────────┐ │ │ │ │ │ │ │ Riyadh Dammam Macca │ │ ┌───┐ ┌──┐ ┌──┐ │ │ A B C D E F │ │ A1 B1 │ └─────────────────────────────────────────┘

Forest Root Domain

أول Domain يتم إنشاؤه داخل Forest. في مثالنا: test.local هو Forest Root Domain.

العلاقة الهرمية

Forest ← Forest Root Domain ← Child Domains ← Resources

03

Parent Domain و Child Domain

Parent Domain

الدومين الرئيسي الذي يحتوي على الدومينات الفرعية. في مثالنا: test.local هو Parent Domain.

Child Domain

دومين فرعي يتبع الـ Parent Domain. يمكن أن يحتوي على فروع أخرى.

test.local │ ├── riyadh.test.local │ ├── A │ └── B │ ├── dammam.test.local │ ├── C │ └── D │ └── macca.test.local ├── E └── F
Child Domains: riyadh.test.local dammam.test.local macca.test.local
04

PDC و ADC

🔵 PDC — Primary Domain Controller

أول Domain Controller يتم إنشاؤه في الدومين

  • يحمل نسخة كاملة من قاعدة بيانات Active Directory
  • هو المصدر الرئيسي للمزامنة
  • يدير العمليات الحرجة مثل FSMO Roles

🟠 ADC — Additional Domain Controller

Domain Controller إضافي داخل نفس الدومين

  • نسخة احتياطية من Active Directory
  • توزيع حمل تسجيل الدخول
  • زيادة الاعتمادية (High Availability)
  • استمرار العمل عند تعطل PDC
test.local │ ├── PDC ← Primary (الأول) ├── ADC1 ← Additional ├── ADC2 ← Additional └── ADC3 ← Additional

مزامنة تلقائية

جميع Domain Controllers تعمل معاً وتقوم بمزامنة بيانات Active Directory تلقائياً عبر replication.

05

خطوات إنشاء الدومين

1

تثبيت Windows Server

تثبيت نظام Windows Server على الجهاز المخصص ليكون Domain Controller.

2

تثبيت دور AD DS

تثبيت دور Active Directory Domain Services (AD DS) عبر Server Manager.

3

ترقية الخادم إلى Domain Controller

تنفيذ Promote this server to a Domain Controller لبدء عملية الإعداد.

4

إنشاء Forest Root Domain

إنشاء Forest جديد وتحديد اسم الدومين مثل: test.local

5

إضافة Additional Domain Controllers

إنشاء ADCs عند الحاجة لتوزيع الحمل وزيادة الاعتمادية.

6

ضم الأجهزة إلى الدومين

Join Domain — ضم أجهزة المستخدمين والخوادم إلى الدومين للإدارة المركزية.
